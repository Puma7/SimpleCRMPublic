## Findings considered and rejected (so nobody re-audits them)

- **`node-pop3@0.11` on the POP3 sync path (DEPS-06)**: pre-1.0 dep on a mail-ingestion path, but well-isolated behind a single wrapper (`electron/email/email-pop3-sync.ts`) and no urgent issue — monitor for abandonment; no plan now.
- **Per-request regex route scan (perf angle of DEBT-01)**: matching ~43 anchored regexes on a short path is microseconds versus a DB round-trip — the `mail-routes.ts` issue is maintainability only (plan 014), not performance.
- **Desktop/server mail logic duplication**: largely a non-issue — both editions share pure logic via `packages/core/src/email` (25 import sites); the SQL/JS Message-ID normalization mirror is a documented, test-asserted invariant tied to migration 0025's indexes.
- **Pagination "missing" on mail lists**: not missing — the main list query is a single keyset-paginated query (`limit+1`, `nextCursor`), no N+1 and no `select *`.
- **`packages/svelte-lab` behind `VITE_ENABLE_SVELTE_LAB`**: an isolated, never-rolled-out React-Flow→Svelte-Flow experiment — a keep/kill decision for the maintainer, not an improvement plan.
